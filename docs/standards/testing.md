# Testing Standards

## Testing baseline

- Keep testing practical for MVP.
- Add unit tests for core logic modules.

## Priority unit test areas

- Task generation.
- Input/setting validators.
- Alarm schedule calculation helpers.

## Manual checklist

- Alarm create/edit/delete works.
- Weekday repeat works.
- Alarm toggle works and persists.
- Stop remains locked until tasks complete.
- Quote appears after successful stop.

## Done criteria

- Feature works in primary flow.
- Core logic tests added/updated if logic changed.
- Docs updated if behavior changed.
