# TaskAlarm Docs

Simple product documentation for `TaskAlarm`.

## Goal

Help users wake up by requiring small brain tasks before alarm stop is allowed. After completion, show a random quote of the day.

## What MVP means here

`MVP` means **Minimum Viable Product**: the smallest useful version that solves the main problem well.

## Folder structure

- `product/` - what the app does (purpose, flow, screens, features).
- `standards/` - coding and testing standards.
- `process/` - git process and team workflow.

## MVP focus

- Keep app small and useful.
- Build reusable modules and avoid duplicate logic.
- Keep theme changes centralized from one place.
- Expand later without rewriting core flows.
