# Git Workflow

## Branch naming

Use:

- `ticketId-<feature>`

Examples:

- `ALRM-01_alarmList`

## Commit format

Use:

- `feature: <short result>`
- `bugfix: <short result>`
- `improvement: <short result>`
- `docs: <short result>`
- `chore: <short result>`

## Working process

1. Clarify ticket scope.
2. Implement smallest useful change.
3. Test core behavior.
4. Update docs when needed.
5. Commit with clear message.
